Prompt Documentation

Project Title
AI Code Reviewer (Team Nexora)

Purpose
This document records the major prompts used during the development and execution of the Automated AI Code Reviewer. 

The project uses AI-assisted development and AI-powered code analysis through a GitHub Actions pipeline integrated with the Google Gemini API (`gemini-2.5-flash`).

The documented prompts were used for:
- AI code review generation
- Code diff analysis
- Strict C# architectural evaluation
- GitHub Action orchestration
- System debugging
- Agent workflow design

AI Development Usage
AI coding assistants were used throughout development for:
- Project planning
- GitHub Actions CI/CD development
- Python SDK Integration (google-genai)
- Prompt engineering for the review agent
- API versioning and debugging
- Documentation generation
The development process followed an AI-assisted workflow as required by the challenge guidelines.

Prompt 1: Baseline Review Agent
Objective
Analyze the provided C# git diff and generate a structured code review highlighting standard issues.

Prompt
"You are an expert C# code reviewer. Your job is to analyze the provided git diff (code changes) very carefully. 
Check for the following 3 specific areas:
1. SOLID Principles: Are there any violations?
2. Null-Handling: Is there any risk of NullReferenceException?
3. Async/Await: Is asynchronous code written correctly without blocking threads?

Format your response using clear bullet points and a summary table so developers can understand easily."

Prompt 2: Advanced Senior Review Agent
Objective
Validate and improve the rigor of the review generated by the AI, acting as a strict Senior Software Engineer.

Prompt
"You are a Senior C# Software Engineer and strict Code Reviewer. Your task is to perform a rigorous technical review of the provided git diff. 

Analyze the changes focusing strictly on the following three criteria:
1. SOLID Principles: Identify any architectural flaws, tight coupling, or violations of Single Responsibility, Open-Closed, Liskov Substitution, Interface Segregation, or Dependency Inversion.
2. Memory & Null-Handling: Scan deeply for potential `NullReferenceException` risks, missing null checks, or improper use of nullable reference types.
3. Concurrency (Async/Await): Ensure non-blocking architecture. Flag any `Task.Result`, `.Wait()`, or `async void` (except in event handlers). Ensure `ConfigureAwait` is used where appropriate.

OUTPUT FORMAT:
- Severity Table: Provide a markdown table listing [File, Issue Type, Severity (High/Medium/Low), Short Description].
- Detailed Feedback: Bulleted list explaining the 'Why' and the 'How to fix' with brief code snippet suggestions.
- If the code is perfect, respond simply with: 'Code looks solid! Approved.'"

Prompt 3: Debugging & Pipeline Orchestration
Objective
Assist in resolving pipeline API failures related to SDK versions during the GitHub Actions execution.

Prompt
"Showing error while adding a new file and ai is showing an error like this rectify and tell me what are all the steps" (Followed by providing the error logs: `404 models/gemini-1.5-flash is not found for API version v1beta`)

Agent Loop Design
The project implements an automated pipeline workflow.

Agent 1: Orchestrator (GitHub Actions)
Responsibilities:
- Detect Pull Request events
- Provision isolated environment
- Inject repository secrets securely
Output: Secure Docker Environment

Agent 2: Review Agent (Python + Gemini)
Responsibilities:
- Extract Git Diff using PyGithub
- Connect to Google Gemini
- Perform strict C# analysis based on Prompt Constraints
- Use GitHub API to post feedback as a PR comment
Output: Final Markdown Code Review Comment

Agent Workflow
Pull Request Opened -> Extract Code Diffs -> Review Agent (Gemini) -> Validate Code Architecture -> Format Markdown Output -> Post PR Comment

This architecture demonstrates a seamless CI/CD Agent Loop capability.

Prompt Engineering Techniques Used

Role Prompting
Assigning specific responsibilities to AI agents.
Examples:
- Expert C# Code Reviewer
- Senior C# Software Engineer

Structured Output Prompting
The model is instructed to produce output using a fixed markdown structure.
Benefits:
- Predictable formatting on GitHub Issues
- Clear Severity Tables
- Easier readability for developers

Constraint-Based Prompting
Explicit rules were provided to:
- Focus STRICTLY on 3 areas (SOLID, Null-handling, Async)
- Prevent generating non-actionable fluff
- Return specific approval messages if perfect

Task Decomposition
The overall problem was divided into:
- Fetching the PR diff
- Feeding it securely to the AI
- Parsing the response
- Posting back to GitHub
This improved system reliability.
